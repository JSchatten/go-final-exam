Общие требования
Система представляет собой Телеграм-бота со следующими требованиями к бизнес-логике:

- Извлечение транскрипций аудиозаписей встреч для быстрого просмотра и индексации текста c помощью SaluteSpeech API.
- Получение краткой выжимки из текста с использованием GigaChat API.
- Хранение и индексация полученных транскрипций в базе данных.
- Уточнение конкретных вопросов у GigaChat.

## План реализации

1) Реализация обработчиков для Телеграм-бота.
2) Реализация клиента для SaluteSpeech API.
3) Реализация клиента для GigaChat API.
4) Работа с базой данных.

## Абстрактная схема взаимодействия с системой

Ниже представлена абстрактная бизнес-логика взаимодействия пользователя с системой:

1) Пользователь регистрируется в Телеграм-боте командой /start.
2) Пользователь загружает аудио или голосовые файлы в Телеграм-клиенте боту.
3) Бот автоматически извлекает транскрипции из полученных файлов и выдаёт краткую выжимку. Текст транскрипции сохраняется в базу данных для последующего поиска по ключевым словам.
4) Пользователь задаёт поисковый запрос командой /find ключевое слово и получает список найденных встреч с упоминанием заданных ключевых слов.
5) Пользователь может просмотреть список всех встреч, отсортированных по дате командой /list.
6) Пользователь может просмотреть транскрипцию встречи по id командой /get <id>.
7) Также есть возможность передать текст вопроса ИИ-ассистенту и сразу получить на него ответ командой /chat.

## Telegram бот

1) Зарегистрировать Телеграм-бота по инструкции https://core.telegram.org/bots/tutorial.
2) С помощью библиотеки gopkg.in/telebot.v3 реализовать обработчики сообщений и файлов от бота.
3) Реализовать загрузку файлов по сгенерированной ссылке из телеграм.
4) Обрабатывать запросы от нескольких пользователей.
5) Создать модуль клиента для телеграм.

Телеграм-бот должен предоставлять следующие обработчики:

- OnText – обработчик текста от бота.
- OnVoice – обработчик файла записи голоса.
- OnAudio – обработчик файла аудиозаписи.

## Общие ограничения и требования

- Хранилище данных – PostgreSQL. Структура таблиц, типы и формат хранения — на усмотрение студента.

## Обработка команд от бота

Обработчик текста от бота: OnText.
На вход обработчика передаётся текст сообщения от пользователя.
Команды от бота можно распарсить и выполнить необходимую логику:

- /start – регистрация пользователя – запоминаем его идентификатор.
- /list – список сохраненных встреч.
- /get – получение текста встречи.
- /find – поиск встречи по ключевым словам.
- /chat – запрос к GigaChat.

Результат обработки команд вернуть в ответном сообщении боту пользователю, который делал запрос.
Обработчик записи голоса от бота: `OnVoice`.
На вход обработчика передаётся ссылка на файл, по которой можно загрузить файл для отправки в SaluteSpeech API.
Обработчик аудиофайла от бота: OnAudio.
На вход обработчика передаётся ссылка на файл, по которой можно загрузить файл для отправки в SaluteSpeech API.
С помощью SaluteSpeech API получается транскрипция встречи и сохраняется в базу `PostgreSQL`.
Реализация логики хранения и поиска по ключевым словам — на усмотрение студента.

## SaluteSpeech клиент

1) Пройти регистрацию https://developers.sber.ru/studio/registration по тарифу «Freemium» для физлиц.
2) Получить доступ к API по инструкции https://developers.sber.ru/docs/ru/salutespeech/api/authentication.
3) Реализовать распознавание речи https://developers.sber.ru/docs/ru/salutespeech/rest/post-async-speechrecognition.

Асинхронное распознавание по протоколу HTTP проходит в несколько шагов:

1) Загрузка файла для распознавания.
2) Создание задачи на распознавание.
3) Проверка статуса задачи через пулинг с периодом.
4) Скачивание результата с текстом.

## GigaChat клиент

1) Получить доступ к API по инструкции https://developers.sber.ru/docs/ru/gigachat/individuals-quickstart.
2) Получить ответы модели GigaChat на сообщения.

## Рекомендации по реализациb

- Реализовать очередь обработки запросов от бота через канал.
- Отдавать ответы API в канал очереди.
- Бот принимает ответы API из канала очереди и отправляет пользователю.
- Каждый пользователь получает ответы только на свои запросы.
